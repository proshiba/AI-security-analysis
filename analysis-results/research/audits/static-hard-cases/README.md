# 難解析検体の静的深掘り（2026-07-17）

## 対象範囲と安全性

このファミリー横断バッチでは、過去に解析が難航した80ケースを対象に、
上限付き静的解析、再帰的なアーティファクト解析、ネイティブエントリCFGの
トリアージ、マネージドILのトリアージを再実行しました。80ケースすべての
真正性を確認して解析し、ルートと子を合わせて142レイヤーを調査しました。

投稿検体も復元検体も実行していません。CPU命令のエミュレーション、
URL・IP・C2への接続、復元バイナリの公開成果物ディレクトリへの永続化も
行っていません。永続化した成果物は、無害化済みJSONとMarkdownだけです。

- [機械可読な報告](deep-static-triage.json)
- [生成されたケース別要約](deep-static-triage.md)
- [ワークフロー、根拠基準、コマンド、失敗確認](../../../../analysis-framework/docs/DEEP-STATIC-ANALYSIS.md)

## 検証済み結果

| 指標 | 結果 |
| --- | ---: |
| インベントリのケース数 | 80 |
| 解析済み | 80 |
| 部分解析／未検出／入力エラー | 0／0／0 |
| ルートおよび子レイヤー | 142 |
| Protector／難読化マーカーを観測したケース | 16 |
| 予算上限に達したケース | 0 |
| 想定した子が不足したケース | 1 |

予算上限に達していないことは、設定済みの走査上限に到達しなかったことを
意味します。レイヤー総バイト予算は、キューへ受け入れた重複しないレイヤー
ペイロードの合計であり、ピークメモリ上限ではありません。また、
Parser／Unpackerの一時割り当ては含みません。この結果は、すべてのProtectorを
除去したこと、すべての暗号化バッファを復号したこと、またはすべての終端
ペイロード／設定を復元したことを意味しません。

## 復元関係

解析した142ノードの内訳は、ルート80件と静的に復元した子62件です。

| 関係 | レイヤー数 | 解釈 |
| --- | ---: | --- |
| `root` | 80 | 真正性を確認したインベントリ入力 |
| `pe-overlay-rar` | 3 | 大容量PEルート内で識別したRARオーバーレイ |
| `pe-resource-opaque` | 12 | ハッシュ化し、データとして再トリアージした不透明PEリソースBLOB |
| `pe-resource-pe` | 2 | PEリソースから復元したPEの子 |
| `pe-inflated-gap-removed` | 1 | 膨張したルートから構造的に縮小して得たPEの子 |
| `dotnet-resource-opaque` | 36 | ハッシュ化したデータレイヤーとして保持したマネージドマニフェストリソース |
| `dotnet-bitmap-rgb-pe` | 5 | マネージドBitmapのRGBデータから復元したPEの子 |
| `embedded-pe` | 3 | コンテナ／キャリアのルートから復元した埋め込みPEの子 |
| **合計** | **142** | **ルート80件＋子関係62件** |

異なるルートに同一データが埋め込まれている場合、不透明リソースのハッシュは
重複し得ます。重複した**実行可能PE候補の子**は
`6f7a3520fb5a30d1c747e7d232b219c1c97a2270429da7aa1f572ac2c60b28be`
（1,348,488バイト）だけです。この子は、RedLine Stealerの2つのルート
`7fc964fe47e08d13c158f808d0d68f3f2b9341dfb1ba6fb2a48690fe27e22682`
および`7b2a28e5ecbdeb4e608026e8c548ef5f50e4aad5da5ae7bfcc5e9ee05e91e80a`
の両方から復元されました。したがって、2つの報告エントリは独立した
ペイロード亜種2件ではなく、重複しない同一の子1件を示します。

## 旧サイズゲート対象ケース

従来の32MiB超ルートに対するスキップを、インベントリ内の6ケースすべてで
廃止しました。現在は、ルート全体をコピーまたは線形走査せずに、範囲を
限定したPE構造解析と到達可能CFG解析を行い、復元した子を別途キューへ
登録します。

| ファミリー | ルート数 | 静的解析結果 |
| --- | ---: | --- |
| QuasarRAT | 3 | 大容量ルート3件を解析し、RARオーバーレイ3件と不透明リソースレイヤー6件を識別 |
| RedLine Stealer | 2 | 両ルートから同一のPEリソース子`6f7a3520...b28be`を取得 |
| HijackLoader | 1 | 膨張したギャップを除去し、PEの子`d11395157a7d11095feeb425d686cf6998530458799785fc00d1ce6f36eef910`を解析 |

「サイズゲート解消」は、これらのルートをスキップしなくなったことを意味します。
RARの内容、Packerロジック、または最終ファミリー設定を例外なく復元したという
主張ではありません。

## マーカーおよびマネージドコードの所見

観測したマーカー文字列の分布は、KoiVM 10件、SmartAssembly 2件、
Themida 1件、UPX 1件、Enigma 1件、nsPack 1件でした。マーカーを含む
ケースは合計16件です。マーカーはルーティングの手掛かりであり、
Protectorのバージョン、Unpackの成功、またはファミリー帰属を証明しません。

マネージドILのトリアージは23レイヤーを対象としました。Vidar／KoiVMの
インベントリ10ケースすべてでKoiVMのルーティング手掛かりが得られました。
これは意図的に`koi_vm: suspected`と報告しています。OldRodまたは別の
バージョン固有手順によって、すべてのメソッド、文字列、リソース、または
改変KoiVM亜種を再構築できることを証明するものではありません。

マネージドレイヤー5件では、1つ以上のメソッドに裏付けのある高Fan-out
Switchが含まれたため、`managed_control_flow_flattening: suspected`を
記録しました。内訳は、njRATのルート1件と子1件、Snake Keyloggerの子2件、
VenomRATの子1件です。これらはマネージドCFGレビューの手掛かりであり、
Flatteningの確認結果ではありません。この5件はVidar／KoiVMの手掛かり
10件とは重ならず、両分類は独立しています。マネージドリソース難読化の
トリアージ結果は、11レイヤーが`suspected`、4レイヤーが`inconclusive`、
8レイヤーが`not_observed`でした。

## ネイティブ制御フローの解釈と誤検知対策

典型的なネイティブ制御フローFlatteningは**未確認**です。ネイティブ
Heuristicから低確度の`suspected`行が2件得られましたが、両方とも
重複した同一のRedLineの子`6f7a3520...b28be`です。範囲を限定した
エントリCFGはDispatcherに似た形状を持ちますが、Dispatcherの状態と
元の後続先対応は復元できていないため、確認基準を満たしません。

結果には、以下の交絡要因補正を反映しています。

- マネージドPEレイヤー13件は、小さなネイティブCLRエントリThunkだけを
  公開しています。このため、ネイティブCFF、間接フロー、Anti-disassembly、
  VM／Protectorの評価は`not_evaluable`とし、代わりにマネージドILへ
  ルーティングしています。
- ネイティブCFF評価5件と間接フロー評価4件は、Packer／Protector／
  コンテナ／ローダー構造でCFG形状を説明できるため`confounded`としました。
  これらの状態は5つのルートにまたがりますが、陽性検出ではありません。
  StealCのルート1件には、間接フローの`suspected`ルーティング手掛かりが
  残りますが、15件の間接転送だけでは難読化やペイロード復元を証明できません。
- UPX、Themida、nsPack、Enigma、SmartAssembly、KoiVMの文字列だけを
  難読化解除成功の根拠として扱うことはありません。
- `not_observed`は、範囲を限定した到達可能エントリCFGがHeuristicを
  満たさなかったことだけを意味します。Callback、未訪問関数、復元した子、
  または利用できないコードを含めた全体的な不在を立証しません。

## 想定した子に残る不足

想定した子が不足しているのは、RemcosRATのルート
`78b21599a83dbfad39c17202d37dd2b6d552c9679755bc199a9826f3dd0e40db`
だけです。インベントリでは、生の中間レイヤー
`e9ed0be544b08189ceca2ec8e6ae8f74d62335ed006f0b207fb211df6bbdcb3a`
を想定していましたが、このバッチでは再現できませんでした。走査予算は
枯渇していません。したがって、Extractor／NSIS／生Decoderの検証不足が
残っています。過去の子が存在しなかったことや、ルートが無害であることを
示す根拠ではありません。

## 解釈の境界

この結果により、静的解析の優先順位付けとレイヤーの追跡可能性が向上します。
Themida／WinLicense、KoiVM、VMProtect、Enigma、nsPack、UPX／MPRESS、独自State
machine、または暗号化ペイロードバッファに対する汎用静的Unpackerを
実現したという主張ではありません。復元を確認済みとするには、一貫した子の形式、
新たに真正性を確認したハッシュ、再現可能なDecoderまたはDispatcherの証明、
および復元レイヤー上でのファミリー／設定検証が必要です。
