# Trivy / TeamPCP 供給網侵害

## 確認済みの範囲

アクアの注意喚起では、漏えいした認証情報を使い、2026年3月19日に悪性の Trivy `v0.69.4` を公開し、`trivy-action` の77個中76個のタグを強制更新して、`setup-trivy` の全7タグを置き換えたことが確認されている。2026年3月22日には、悪性のドッカーハブイメージ `0.69.5` と `0.69.6` も公開された。

公開されていた時間帯は協定世界時で、トリビーv0.69.4が約3時間、アクションが約12時間、セットアップ用アクションが約4時間、ドッカーハブの0.69.5および0.69.6が約10時間である。悪性コードはトリビーの主ブランチには存在しなかったが、生成物がキャッシュに残っている可能性がある。

## マルウェアの挙動

- リリースコミット `1885610c` は `actions/checkout` を偽装コミット `70379aad` へ置き換え、検証を回避した
- アクションのペイロードは `Runner.Worker` のメモリを `/proc/<pid>/mem` を介してダンプし、50か所を超える秘密情報の保存場所を走査した
- 収集内容は持ち出し前に AES-256-CBC と RSA-4096 で保護された
- 主インフラ: `scan.aquasecurtiy.org` および `45.148.10.212`
- 予備経路: 公開 `tpcp-docs-*` リポジトリを作成し、`data-<timestamp>` というリリース生成物をアップロードする。これは `INPUT_GITHUB_PAT` を利用できる場合に実行される

報道されたシスコのソースコード窃取は本事案の下流で発生した可能性がある。しかし、報道されたリポジトリ数、ソースの範囲、窃取された AWS 鍵を確認するシスコの一次情報は見つからなかった。これらの詳細は `unverified/reporting-only` のままであり、Aqua が確認した範囲へ混在させてはならない。

## 選別調査と検出

1. 公式に示された公開時間帯に含まれるワークフロー実行を特定する。
2. 実際に参照したアクションのコミットと取得したイメージのダイジェストを解決する。現在の変更可能なタグが指す値だけに依存しない。
3. 公開されたランナーからアクセス可能だった秘密情報をすべて侵害済みとみなし、不可分な手順で更新する。
4. 組織の監査ログで、公開 `tpcp-docs-*` リポジトリ、リリース作成、`data-*` 生成物を検索する。
5. ランナーのネットワーク遠隔測定情報で、綴りを偽装した接続先または IP への通信と、アクション用シェルによる `/proc/*/mem` アクセスを調査する。

誤検出評価:

- **低い**: 悪性生成物またはイメージの完全一致ダイジェスト、綴りを偽装した接続先、予期しない `tpcp-docs-*` のリリース生成物。
- **中程度**: 公開時間帯に実行された、変更可能な古いアクションまたはセットアップ用アクションの参照。現在のタグだけでは過去の実行を証明できない。
- **高い**: ランナー、プロセス、時刻の相関を伴わない、認証情報ファイルまたは `/proc` への一般的なアクセス。

`analysis-framework/common/supply_chain_audit.py` を使用すると、マニフェストとワークフローの根拠をオフラインで検索できる。ルール: [Sigma](rules/trivy_teampcp.yml)、[YARA](rules/trivy_release_hashes.yar)、[IOC](iocs.json)。
